## Using Discussions

The goal here is to manage khaaliDimaag using github discussions (and github in general) while also
having enabling public transparency and input. As such, the discussion categories are designed to 
help move a _thought-bubble_, all the way to a _release-announcement_, going through all manner of 
discussions from implementation details to ethical concerns to business impact.

**Main Idea ::** A `0|1` coupling system where, \
`0` == heart / abstract / discover / reveal \
`1` == brain / specific / show-off / flex

There are ~16 steps an idea must take from inception to public release, represented by 4 _bits_. 
These "bits" also inform the progression of the idea towards a full release. Each place value has 
a special meaning for the face value of the bit,
- `x...` discussion (0) vs implementation (1)
- `.x..` philosophical (0) vs "tethered" (1)
- `..x.`
- `...x`

So, a _philosophical discussion (00xx)_ is going to be a pure idea exploration, as compared to a 
_philosophical implementation (10xx)_ which will be revolve more around the idea's practicality. 
Similarly, a _tethered discussion (01xx)_ will be about "feeling" the idea in the real world, while
a _tethered implementation (11xx)_ will tackle the business case of the idea.

With this, we can now define the [16] discussion categories, in order of progression from inception
to public release,
